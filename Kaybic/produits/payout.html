Comment fonctionne l'API de décaissement

Envoyez des fonds instantanément vers n'importe quel compte mobile money en Afrique de l'Ouest avec un simple appel API.

1

Initialiser le décaissement

Envoyez une requête API avec les informations du destinataire (numéro de téléphone, montant, opérateur) pour initier le transfert vers un portefeuille mobile.

Autorisation opérateur

3

Réception des fonds

Le destinataire reçoit instantanément les fonds sur son portefeuille mobile money. Vous recevez une confirmation avec le statut de la transaction en temps réel.

import requests
import json

# Configuration de l'API KAYBIC AFRICA
api_key = "votre_cle_secrete"
api_url = "https://api.kaybic.africa/v1/payouts"

# En-têtes de la requête
headers = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json"
}

# Données de la demande de paiement
payload = {
    "amount": 5000,               # Montant en FCFA
    "phone": "22507223344",       # Numéro du destinataire
    "operator": "orange-money",   # orange-money, mtn, moov, wave
    "reference": "PAY-12345",     # Votre référence unique
    "description": "Paiement fournisseur mai 2025"
}

try:
    # Création d'un paiement sortant
    response = requests.post(api_url, headers=headers, json=payload)
    response.raise_for_status()
    
    # Traitement de la réponse
    result = response.json()
    print(f"Paiement créé! ID: {result['id']}")
    print(f"Statut: {result['status']}")  # En attente, Traité, Échoué
    
except requests.exceptions.RequestException as e:
    print(f"Erreur: {e}")


// Installation: npm install axios

const axios = require('axios');

// Configuration de l'API KAYBIC AFRICA
const apiKey = 'votre_cle_secrete';
const apiUrl = 'https://api.kaybic.africa/v1/payouts';

// Données du paiement
const payoutData = {
    amount: 5000,                // Montant en FCFA
    phone: '22507223344',        // Numéro du destinataire
    operator: 'orange-money',    // orange-money, mtn, moov, wave
    reference: 'PAY-12345',      // Votre référence unique
    description: 'Paiement fournisseur mai 2025'
};

// Création d'un paiement sortant
axios.post(apiUrl, payoutData, {
    headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
    }
})
.then(response => {
    const result = response.data;
    console.log(`Paiement créé! ID: ${result.id}`);
    console.log(`Statut: ${result.status}`); // En attente, Traité, Échoué
})
.catch(error => {
    console.error('Erreur:', error.response ? error.response.data : error.message);
});

Questions fréquentes

Tout ce que vous devez savoir sur notre API de décaissement mobile money.

Les décaissements sont généralement instantanés (quelques secondes). Dans certaines situations exceptionnelles, comme lors de maintenance des opérateurs ou en cas de fort volume, le délai peut s'étendre jusqu'à 10 minutes. Notre API vous permet de suivre en temps réel le statut de chaque transaction et de configurer des notifications automatiques pour être informé dès que le statut change.

Vous disposez de plusieurs options pour approvisionner votre compte KAYBIC AFRICA :

  1. Virement bancaire : Effectuez un virement vers notre compte bancaire local dans votre pays.
  2. Mobile money : Alimentez votre compte directement depuis un portefeuille mobile money.
  3. Carte bancaire : Rechargez instantanément votre compte via carte bancaire (Visa, Mastercard).

Pour les clients Business et Enterprise, nous proposons également la possibilité d'opérer à crédit après validation de votre dossier.

Notre API de décaissement couvre actuellement 8 pays de l'Afrique de l'Ouest :

  • Côte d'Ivoire (Orange Money, MTN Mobile Money, Moov Money, Wave)
  • Sénégal (Orange Money, Wave, Free Money)
  • Mali (Orange Money, Moov Money)
  • Burkina Faso (Orange Money, Moov Money)
  • Bénin (MTN Mobile Money, Moov Money)
  • Togo (Flooz by Moov)
  • Niger (Orange Money, Moov Money)
  • Guinée (Orange Money, MTN Mobile Money)

Nous ajoutons régulièrement de nouveaux pays et opérateurs. Consultez notre documentation pour la liste à jour.

Pour les paiements récurrents, notre API offre deux options :

  1. Planification programmée : Configurez des paiements automatiques à intervalles réguliers (quotidien, hebdomadaire, mensuel).
  2. API de paiements en masse : Envoyez un fichier CSV contenant jusqu'à 10 000 destinataires pour traitement simultané.

Ces fonctionnalités sont disponibles pour les clients Business et Enterprise. Notre dashboard vous permet de suivre tous vos paiements programmés et d'obtenir des rapports détaillés sur chaque transaction.

Les limites de transaction dépendent des réglementations locales et des politiques des opérateurs mobile money dans chaque pays. En général :

  • Limite par transaction : Entre 500 000 et 2 000 000 FCFA selon les pays.
  • Limite quotidienne : Entre 2 000 000 et 10 000 000 FCFA selon les pays.
  • Limite mensuelle : Entre 10 000 000 et 30 000 000 FCFA selon les pays.

Pour les clients Enterprise, nous pouvons mettre en place des arrangements spéciaux avec les opérateurs pour augmenter ces limites. Contactez notre équipe commerciale pour plus d'informations.
